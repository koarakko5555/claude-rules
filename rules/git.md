# Git 規約

## コミット

- コミットメッセージは命令形・件名 ≤72 文字。WHAT より WHY を書く。
- amend より新規コミットを優先する（公開済みコミットの amend は履歴を壊す）。
- `--no-verify` でフックをスキップしない（明示的に指示された場合を除く）。
- main / master への force push は禁止。
- ステージングはファイル名を指定する。`git add -A` はシークレット混入リスクがあるため避ける。

## ブランチ

- main から feature ブランチを切り、PR でマージする。
- ブランチ名: `<type>/<short-description>` 形式（例: `fix/auth-redirect`, `feat/dark-mode`）。

## Pull Request

- PR タイトルは ≤70 文字。
- 本文: サマリ箇条書き + テストプラン（チェックリスト形式）。
- **PR はできる限り細かく切る**。1 PR = 1 論理変更、理想は ≤300 行。
- 機能単位で 1 つの大きな PR にせず、責務（スキーマ / API / UI / 配線 など）で分割する。
- 各 PR は単独でマージしても既存機能を壊さないこと。半端な状態は feature flag か未配線で隠す。
- 分割すると逆に意味が壊れる、または 50 行未満で完結する場合のみ単一 PR を許容。
